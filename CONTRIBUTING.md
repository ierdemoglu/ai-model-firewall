# Contributing Guide

Thank you for considering contributing to the AI Model Firewall project! This guide outlines the rules and standards to ensure smooth and consistent contributions.

## 📋 Contribution Rules

### 1. Code Standards
- Follow **PEP 8** Python coding standards.
- Use clear and descriptive function and variable names.
- Write **docstrings** for every new module.

### 2. Adding a New Security Module
If you want to add a new security check:

1. Inherit from the `BaseSecurityModule` class in `modules/base_module.py`.
2. Implement the `name` property and `scan()` method.
3. Register your module in `core/pipeline.py`.
4. Write appropriate tests.

**Example:**
```python
from .base_module import BaseSecurityModule

class MyNewModule(BaseSecurityModule):
    @property
    def name(self) -> str:
        return "My New Module"

    def scan(self, file_path: str) -> dict:
        # ... security check ...
        return {"success": True, "message": "Clean"}
```

### 3. AI Usage Disclosure
If your Pull Request contains code generated by **AI tools** (Copilot, ChatGPT, Gemini, etc.), please honestly disclose it in the PR description.

Example:
> "The core logic of the `DiskSpaceGuardModule` in this PR was developed with AI assistant support."

### 4. License and Copyright
All submitted code will be evaluated under the **Business Source License 1.1**. Commercial rights are protected.

### 5. Pull Request Process
1. Create a new branch from `main` (e.g., `feature/new-module`).
2. Commit your changes.
3. Open a Pull Request with a clear title and description.
4. Obtain at least one maintainer approval.

---

**Thank you!** Every contribution is valuable.
