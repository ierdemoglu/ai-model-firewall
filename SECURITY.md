# Security Policy

We take the security of this project very seriously. Responsible disclosure of any security vulnerabilities found in AI Model Firewall is critical.

## Supported Versions

Only the latest version on the `main` branch receives security support.

| Version | Support Status      |
|---------|---------------------|
| main    | ✅ Active support    |
| < main  | ❌ Not supported    |

## Reporting Procedure

**Please do not report security vulnerabilities through public GitHub Issues.**

Instead, please follow these steps:

1. **Email the Report**: Send an email to `security@aimodelfirewall.dev`.
2. **Subject Line**: `[SECURITY] Brief Description`
3. **Content**:
   - Detailed description of the vulnerability
   - Affected files and functions
   - Proof of Concept (PoC) code if possible
   - Expected impact level (Critical / High / Medium / Low)

**Response Time**: You will receive a response within 48 hours. Critical vulnerabilities will be addressed faster.

## Coordinated Disclosure

After the vulnerability is fixed, it will be publicly disclosed in coordination with the reporter.

---

*This policy follows best practices for open-source cybersecurity projects.*
